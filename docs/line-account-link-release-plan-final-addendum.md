# LINE自己解決型アカウント連携：改訂リリース実行計画への最終追加指示

## 判定

改訂計画は承認ゲート、秘密情報の扱い、本番コードデプロイの順序を満たしている。

以下の修正を反映した後、フェーズ1の隔離DBテストから実行してよい。本番DBマイグレーションは、計画どおりユーザーの明示承認まで実行しないこと。

## P1：Webhook到達性確認の方法を修正する

フェーズ7の「`/api/webhooks/line` がHTTPSで200応答すること」は、通常のブラウザアクセスや未署名のHTTPリクエストでは確認しないこと。

このエンドポイントはLINE署名付きのPOST専用であり、未署名リクエストは401、GETは405などになることが正常である。

フェーズ7は以下に置き換える。

1. デプロイ先の `/line/link` を匿名ブラウザで開き、ログイン画面または連携画面が表示されることを確認する。
2. LINE Developers ConsoleのWebhook URL検証機能で `https://<本番ドメイン>/api/webhooks/line` を検証し、成功することを確認する。
3. 実機でLINE公式アカウントへ「連携」と送信し、Webhook起点のReply API応答が返ることを確認する。
4. 署名不正時の401は自動テストで確認済みとして記録し、公開環境へ未署名リクエストを送って確認しない。

## P1：デプロイ対象リビジョンを固定する

フェーズ2で検証したコードと、フェーズ6でデプロイするコードが同一であることを記録する。

- テスト成功後にコミットを作成し、コミットSHAを記録する。
- 本番デプロイはそのコミットSHAを含むブランチまたはリリースタグから行う。
- デプロイ後に、デプロイ識別子・コミットSHA・環境名を完了報告へ記載する。
- テスト後に機能コードを変更した場合は、フェーズ2の検証を再実行する。

## P1：環境変数の設定範囲を確認する

`NEXT_PUBLIC_*` はビルド時にクライアントバンドルへ反映されるため、フェーズ6のデプロイ前に設定を完了すること。

- Vercel等の環境変数は、対象となるProductionまたはPreview環境へ設定する。
- ステージングを利用する場合、ステージングの公開URLに対応する `NEXT_PUBLIC_LINE_CONNECT_URL` 等を設定する。
- 環境変数を変更した後は、対象環境を再デプロイする。
- `LINE_CHANNEL_SECRET`、`LINE_CHANNEL_ACCESS_TOKEN`、`POSTGRES_URL` をクライアント公開変数にしない。

## P2：公開後監視の確認先を具体化する

追加の有料監視サービスは導入しない。

- デプロイ先の既存ログ画面で、個人情報を含まない `line_account_link` 構造化ログを確認する。
- 確認するのはイベント名、成功可否、固定エラーカテゴリ、処理時間のみとする。
- 連携失敗が発生した場合も、LINE user ID、nonce、link token、メールアドレス、例外本文を調査ログへ貼り付けない。

## フェーズ1開始の条件

ローカルDockerの専用DB `logeats_test_line_link_verify` は構築済みである。以下のいずれかを確認してから、隔離DBテストを開始すること。

- Dockerコンテナ `logeats-test-pg` が起動している。
- テストDB名が `logeats_test_line_link_verify` である。
- `TEST_POSTGRES_URL` はローカルPowerShellプロセスにだけ設定され、表示されていない。

隔離DBテストがExit code 0で成功した時点で、フェーズ2へ進むこと。

## P1：コミットSHAの固定は本番マイグレーションより前に行う

改訂リリース計画では、コミット作成がフェーズ6（本番コードデプロイ）に置かれているが、これはフェーズ5の本番DBマイグレーションより後であり、検証済みリビジョンを固定する要件を満たさない。

以下の順序へ修正すること。

1. フェーズ2のローカル検証が成功する。
2. 今回の機能変更とリリース手順書だけを確認してコミットし、コミットSHAを記録する。
3. ユーザーの明示承認後、そのコミットSHAを対象に本番DBマイグレーションを実行する。
4. 同一コミットSHAを本番へデプロイする。

コミット前に、意図しないファイル、生成物、環境変数ファイルが含まれていないことを `git status --short` と `git diff --cached --check` で確認すること。コミットまたは本番DBマイグレーションは、ユーザーの権限・承認方針に従って実行すること。
