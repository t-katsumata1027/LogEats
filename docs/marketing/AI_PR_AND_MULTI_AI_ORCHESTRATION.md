# 人物型AI広報・SNS・マルチAIオーケストレーション

更新日: 2026年7月23日

## 1. 人物型AI広報の方針

### 主役

人物型の架空AI広報をブランドの主役にする。

- 仮名: **みな｜Log-Eats AI広報**
- 属性: 完全な架空人物
- 役割: フードテックスタートアップの清潔感ある広報、食事記録の伴走者
- 見た目: 実在するスタートアップ広報のように自然だが、実在人物を模倣しない
- 専門性: 栄養に関する一般情報を一次資料に基づいて分かりやすく伝える
- 禁止: 栄養士、管理栄養士、医師などの資格保有を事実なしに名乗らない

## 2. 透明性ルール

「プロフィールを開かなければ分からない」状態に依存しない。

必須表示:

1. 表示名: `みな｜Log-Eats AI広報`
2. bio: AIエージェントであることを明記
3. 固定投稿: 人物、画像、文章がAI生成を含み、運営が確認することを説明
4. 健康・広告投稿: 投稿本文にも必要な注意と広告表示

bio案:

> Log-Eats公式AI広報のみなです。食事写真からカロリー・PFCの目安を記録するサービスを紹介します。投稿はAIが下書きし運営が確認。解析値は推定で、医療・栄養指導を目的としません。

固定投稿案:

> はじめまして。Log-Eats公式AI広報の「みな」です。私は実在人物ではなく、AIで作られた広報エージェントです。サービスの使い方、食事記録を続ける工夫、開発の裏側を、運営チームの確認を経てお届けします。カロリーやPFCは目安として扱い、診断や個別の栄養指導は行いません。

アフィリエイト投稿は、冒頭など最初に認識できる位置へ`広告`、`PR`、`アフィリエイトリンクを含みます`を表示する。ハッシュタグ末尾だけに隠さない。

## 3. 人格と口調

- 一人称: 私
- 文体: 短く、明るく、知的、押しつけない
- 基本姿勢: 食事や体型を評価・叱責しない
- 推奨表現:
  - 「写真から目安を見てみましょう」
  - 「完璧より、記録を続けられたことが一歩です」
  - 「量や調理法で推定値は変わります」
- 禁止表現:
  - 「これで必ず痩せます」
  - 「この食事は悪です」
  - 「栄養士の私が保証します」
  - 「病気を予防・治療できます」
  - 根拠のない精度、人数、受賞、口コミ

## 4. コンテンツピラー

| ピラー | 比率 | 内容 |
|---|---:|---|
| 使って分かる | 30% | 実画面で写真・テキスト解析を15〜30秒で見せる |
| 続ける工夫 | 20% | 忙しい日、入力忘れ、LINEとの使い分け |
| 食事データの読み方 | 20% | PFC、カロリー、誤差を一次資料付きで解説 |
| 参加型 | 15% | 投票、質問、機能要望、解析クイズ |
| プロダクト透明性 | 10% | 苦手な例、改善、データソース、プライバシー |
| 収益コンテンツ | 5% | 適合する案件のみ。広告表示と実使用根拠を付ける |

各投稿には次のどれか1つ以上の固有価値を必須とする。

- 実際のアプリ画面
- Log-Eats独自の実測
- 十分に匿名化した集計
- 許諾済み素材
- 運営の一次経験
- 公的一次資料に基づく独自解説

## 5. 週間投稿テンプレート

| 曜日 | 投稿 |
|---|---|
| 月 | 15秒サービスデモ。CTAは無料解析 |
| 火 | PFCまたは記録の1テーマ解説。出典付き |
| 水 | 投票・質問募集。売り込みなし |
| 木 | 解析が難しい例、誤差、改善の透明性 |
| 金 | 習慣化の小技、週末チャレンジ |
| 土 | 許諾済みUGCまたは運営自身の利用例 |
| 日 | 週次改善報告。適合案件がある場合のみ収益投稿 |

開始量:

- X: 1日1投稿を上限
- Instagram Reels: 週3本を上限
- 返信: 人間確認済みのみ
- 同じ動画でも、媒体ごとに冒頭、字幕、CTAを調整する

## 7. 2026年7月23日時点のモデル選定

モデルは「最新だから」ではなく、役割、品質、料金、提供状態で選ぶ。実行前に公式Models APIと公式モデルページを再確認する。

### Codex司令塔

| 用途 | 採用モデル | 理由 |
|---|---|---|
| 日次集計、優先順位、brief、PR作成 | `gpt-5.6-terra` | 品質と費用のバランス |
| 週次深掘り、重要リリース、事故分析 | `gpt-5.6-sol` | 複雑な推論とコーディング |
| 大量の整形、分類、重複除去 | `gpt-5.6-luna` | 高頻度処理の費用を抑える |

OpenAI公式モデル一覧では、GPT-5.6 Solは複雑な専門作業、Terraは知能と費用の均衡、Lunaは費用重視の大量処理向けとされている。

### Gemini担当

採用: `gemini-3.6-flash`

Google公式情報では2026年7月21日公開の安定版で、本番利用可能。テキスト、画像、動画、音声、PDF入力、約100万トークン入力、構造化出力、関数呼出し、検索グラウンディング、思考、コード実行を扱える。

担当:

- 食事・アプリ画面・縦動画のマルチモーダル確認
- 投稿候補と媒体別フック
- 大量の検索意図・コメント分類
- 検索グラウンディングを使う一次調査
- 低遅延の反復タスク

### Claude担当

| 用途 | 採用モデル | 理由 |
|---|---|---|
| 通常の長文編集、ブランド口調、レッドチーム | `claude-sonnet-5` | 速度と知能の組み合わせ |
| 健康・広告・危機対応など高リスクの上位審査 | `claude-fable-5` | Anthropicの広く提供される最上位モデル |

担当:

- 記事と投稿の論理・読みやすさ
- 誇大な含意や資格誤認の検出
- 反対意見からのレッドチーム
- 広告表示、健康表現、権利の審査

### 画像生成

- 人物型AI広報と補助素材はGPT Image 2相当の最新画像モデルを候補とする。
- 同一人物の継続性を評価し、基準画像、服装、照明、禁止項目を固定する。
- 実在人物の模倣、著名人風、資格の偽装を禁止する。

## 9. モデルレジストリ

コードに単一モデル名を散在させず、役割単位で設定する。

例:

```yaml
roles:
  orchestrator_daily:
    provider: openai
    model: gpt-5.6-terra
  orchestrator_high_risk:
    provider: openai
    model: gpt-5.6-sol
  multimodal_worker:
    provider: google
    model: gemini-3.6-flash
  editorial_reviewer:
    provider: anthropic
    model: claude-sonnet-5
  high_risk_reviewer:
    provider: anthropic
    model: claude-fable-5
```

実行前チェック:

1. Models APIでモデルIDが利用可能か確認
2. previewではなく安定版か確認
3. 廃止予定日を確認
4. 入出力モダリティと構造化出力を確認
5. 料金とレート制限を確認
6. 小さな評価セットを実行
7. 合格時だけレジストリを更新

モデル変更は自動適用せず、評価結果をDiscordへ送り、人間承認後に切り替える。

## 11. 品質ゲート

1つでも失敗すれば公開しない。

1. Grounding: 数値、仕様、価格、効果に根拠がある
2. Product truth: 現行実装と一致する
3. Health safety: 診断、治療、保証、極端な制限を含まない
4. Credential truth: 資格・監修・専門家を偽装しない
5. Commercial disclosure: 広告・報酬関係が一見して分かる
6. Rights/privacy: 素材、UGC、個人情報、位置情報の権利を確認
7. Original value: 微差量産ではなく固有価値がある
8. Measurement: UTM、creative_id、CTA、成功条件がある
9. Human approval: 初期30日と高リスク投稿は承認記録がある

## 12. 自動停止条件

- AIである旨、広告表示、推定値の注意が欠落
- 無根拠な健康効果、資格、人数、口コミ
- 個人情報、未許諾UGC、実在人物への過度な類似
- 存在しない機能や価格の宣伝
- LPの4xx/5xx
- GA4イベント、UTM、creative_idの欠損
- 解析成功率または結果表示速度の急激な悪化
- 苦情や訂正率の急増

停止時は予約投稿を止め、原稿、検出理由、根拠をDiscordへ通知する。AIが自動で謝罪や反論を公開せず、人間責任者へ引き上げる。
