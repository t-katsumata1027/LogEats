# 検討中機能・今後の改善案（LogEats）

競合アプリ（あすけん、カロミル、FoodLog、カルシー）との比較分析に基づき、LogEatsの今後の差別化および機能向上のために検討している機能一覧です。

## 1. データ精度とデータベースの網羅性の向上
現状のLLMによる推論（ハルシネーションの可能性あり）を補完し、より正確なカロリー・栄養素の記録を実現するための機能。

*   **ハイブリッド画像認識（OCR + DB照合）**
    *   画像から商品名やパッケージのテキストをOCR抽出し、既存または外部APIの食品データベース（例：楽天レシピAPIなど）と照合する仕組み。
    *   *※現在開発中の「コンビニ商品の正確な検索」機能はこの一環。*
*   **バーコードリーダー（Web API）の実装**
    *   Webカメラ等を利用したバーコードスキャン機能。正確な市販品データの入力経路として提供。

## 2. 栄養分析の深さとパーソナライズ
ただマクロ栄養素を計算するだけでなく、ユーザーの健康管理をより深くサポートするための機能。

*   **詳細な微量栄養素のトラッキング**
    *   PFC（タンパク質・脂質・炭水化物）とカロリーに加え、塩分、食物繊維、ビタミンなどの推移も追跡・表示。
*   **AIによるパーソナルアドバイス（AIコーチング）**
    *   1日の終わりに「今日の食事サマリーと明日の改善点」を自然言語でフィードバックするダッシュボード上のエージェント機能（例：チャットUIや専用メッセージ領域でのアドバイス）。

## 3. UXと記録の手軽さ
記録の摩擦を極限まで減らし、毎日の継続を容易にするための機能。

*   **PWA（Progressive Web App）化の強化**
    *   「ホーム画面に追加」を促し、ネイティブアプリに近い起動速度とUXを提供。
*   **自然言語（テキスト/音声）による爆速入力**
    *   「おにぎり2個とサラダチキンを食べた」といった自然言語テキストや音声入力をLLMに解析させ、自動でカロリーとPFCを記録する手軽な入力機能。
*   **曖昧な画像のハンドリングとUX向上**
    *   遠景や複数人が写っている写真など、認識が難しい画像の場合にユーザーへ適切にフィードバックする機能。
    *   *※現在実装進行中。*

## 4. モチベーション向上と継続支援（ゲーミフィケーション）
「単なる記録ツール」から脱却し、ユーザーの感情に訴えかけて利用継続を促す機能。

*   **食事スコア・ランク機能**
    *   1日のPFCバランスや目標達成度合いに応じて、目標に対するスコア（例：100点満点）をAIに算出・表示させる仕組み。
*   **連続記録バッジやストリーク（習慣化UI）**
    *   ダッシュボードに「連続記録日数」を視覚化するUI（GitHubのコントリビューショングラフのような形式など）を導入し、途切れさせたくない心理を刺激する。

### 方針・コンセプト
**「巨大な手動データベース」ではなく、「LLMを最大限に活かした超パーソナライズと柔軟で手軽な記録」をLogEatsの最大の差別化ポイントとする。**
